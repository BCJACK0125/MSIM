# MSIM 專案：肥水不落外人田

MSIM (My Shit Is Mine) 是一個自動化交易決策儀表板，旨在幫助使用者在「親友代購」或「二手交易」等情境中，快速計算並比較透過電商平台（如蝦皮）交易與線下面交的成本與收益差異。

專案核心目標是回答一個問題：**「這次交易，我應該請朋友直接匯款，還是讓他去我的賣場下單？」**

透過將複雜的平台手續費、金流處理費、信用卡回饋、以及各種限時活動優惠納入模型，MSIM 提供即時的數據洞察，讓使用者能以最大化自身利益為目標，做出最佳決策。

## 專案架構與技術棧

本專案採用了現代化的 Web 技術與自動化流程，主要包含以下幾個部分：

- **前端介面 (`index.html`)**:
  - 使用 **Tailwind CSS** 打造具備賽博龐克風格的 UI/UX。
  - 採用 **ECharts** 將複雜的數據關係視覺化，繪製出清晰的套利空間分析曲線。
  - 核心運算邏輯完全在前端透過 **JavaScript** 實現，確保了快速的響應與即時互動。
  - 資料與顯示邏輯分離，由 Python 後端動態注入最新的優惠活動資料。

- **後端爬蟲與 AI 分析 (`scraper.py`)**:
  - **`requests`** + **`BeautifulSoup`**: 定期爬取指定電商平台或銀行的優惠活動頁面，獲取最新的原始 HTML 資料。
  - **GitHub Models (AI)**: 利用大型語言模型（如 GPT-4o）的自然語言處理能力，將非結構化的 HTML 活動內容，解析並轉換為標準化的 JSON 格式。
    - **智慧容錯機制**: 當主要 AI 模型失效或返回錯誤時，系統會自動切換至備援模型重試，確保服務穩定性。
  - **數據快照 (`esun_discounts.json`)**: 爬蟲解析後的原始活動資料會被儲存下來，作為 AI 分析的基礎，也方便日後追蹤與除錯。
  - **自動化更新**: 整個爬蟲 -> AI 分析 -> 注入前端的流程，可透過 GitHub Actions 等 CI/CD 工具實現全自動化排程執行。

- **自動化通知**:
  - 爬蟲腳本在完成更新後，會透過 **SMTP** 發送 Email 通知，讓管理者能即時掌握最新的數據同步狀態。

## 核心功能

1.  **參數化模擬**:
    - **買家端**: 可自訂交易金額、信用卡回饋率、回饋上限，並能動態增減、啟用或停用各種由 AI 自動同步的優惠活動。
    - **賣家端**: 可設定不同商品類別的平台抽成費率、賣家身分（影響手續費上限），以及是否參與平台的免運活動。

2.  **即時損益計算**:
    - **買家總收益 (Inflow)**: 精確計算信用卡回饋與所有適用活動疊加後，買家能獲得的總價值。
    - **賣家總成本 (Outflow)**: 綜合考量成交手續費、金流處理費、免運費等所有賣家需承擔的費用。
    - **淨利差 (Spread)**: `買家總收益 - 賣家總成本`，這是決定交易策略的關鍵指標。正值代表「平台套利」成功，負值則意味著線下面交更划算。

3.  **視覺化決策輔助**:
    - **最佳化曲線圖**: 透過 ECharts 動態繪製出在不同交易金額下，淨利差的變化趨勢。圖表中的「死亡交叉點」清晰地揭示了平台交易從有利潤變為虧損的轉折點。
    - **情境模擬按鈕**: 提供「一鍵帶入」常用交易情境（如：高單價 3C 產品、二手車），方便使用者快速開始分析。

## 優化與改進

- **架構統一**: 移除了原有的 Node.js 爬蟲 (`sync-promos.js`)，將後端邏輯完全統一由更適合數據處理與 AI 整合的 Python 腳本執行。
- **資料注入方式**: 改進了 `index.html` 的資料載入機制，從寫死靜態變數，改為由 Python 動態注入全域 `window` 物件。這種方式更靈活、更易於維護，並徹底解決了前後端數據同步的問題。
- **AI 模型穩定性**: 引入了主/備援模型自動切換機制。當主要的 `gpt-4o` 模型因故無法服務時，系統會自動嘗試使用 `gpt-4o-mini` 等備援模型，大幅提升了核心功能的穩定性與可靠性。
- **錯誤處理**: 增強了 HTTP 請求的錯誤處理，當發生 5xx 等伺服器錯誤時，會將 Response Body 的內容記錄下來，便於快速定位問題。

## 如何使用

1.  **設定環境變數**:
    - `GITHUB_TOKEN`: 用於呼叫 GitHub Models API 的金鑰。
    - `GMAIL_USER` / `GMAIL_APP_PASSWORD`: (選用) 設定 Gmail 帳號與應用程式密碼以啟用郵件通知功能。
    - `GITHUB_MODELS_PRIMARY` / `GITHUB_MODELS_FALLBACKS`: (選用) 自訂主要與備援的 AI 模型名稱。

2.  **安裝依賴**:
    ```bash
    pip install requests beautifulsoup4
    ```

3.  **執行更新腳本**:
    ```bash
    python scraper.py
    ```
    腳本執行後，`index.html` 將被自動更新為包含最新優惠活動的版本。

4.  **查看結果**:
    直接在瀏覽器中打開 `index.html` 檔案，即可開始使用儀表板進行分析。
